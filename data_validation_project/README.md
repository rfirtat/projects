# Data Validation Pipeline (Python + Pandas)

## Overview

This project implements a simple **data validation and cleaning pipeline** using Python and pandas.
The goal is to simulate how real-world data pipelines validate, clean, and log incoming data before it is used downstream.

The script reads raw transaction data from a CSV file, applies multiple validation rules, removes invalid records, and writes a cleaned dataset to a new file.

Invalid records are logged to a log file for auditing and debugging purposes.

---

## Features

The pipeline performs the following validation and cleaning steps:

* Remove duplicate transactions based on `transaction_id`
* Ensure `quantity` values are numeric and greater than zero
* Ensure `price` values are numeric and greater than zero
* Validate products against a reference dataset (`valid_products.csv`)
* Validate email format using a regex pattern
* Validate order dates and remove invalid date values
* Standardize and validate state values

Invalid records are removed from the dataset and logged for traceability.

## How the Pipeline Works

The script processes the data using a sequence of validation functions:

```
validate_duplicates()
validate_quantity()
validate_price()
validate_product()
validate_email()
validate_dates()
standardize_state()
```

Each function performs the following steps:

1. Identify invalid records
2. Log invalid rows to the log file
3. Remove invalid records from the dataset
4. Print a summary of how many rows were removed

This pipeline-style design makes the code modular and easy to extend.

---

## Logging

Invalid rows are written to a log file located at:

```
logs/errors.log
```

Example log entry:

```
2026-03-18 18:42:10 - Invalid email removed | email=customer5@email | transaction_id=1005
```

Logging allows invalid records to be reviewed later without interrupting pipeline execution.

## Running the Pipeline

From the project root directory:

```
python src/validation_pipeline.py
```

After execution:

* Cleaned data will be written to:

```
output/cleaned_sales_data.csv
```

* Invalid records will be recorded in:

```
logs/errors.log
```
